## CLI Usage Examples

### Check All Production EDI Errors
```bash
cd plugins/celigo-integration

# Check SPS partners
for id in "${SPS_PARTNERS[@]}"; do
    echo "=== Checking $id ==="
    python3 scripts/celigo_api.py integrations errors "$id"
done
```

### Get Trading Partner Flows
```bash
# Academy flows
python3 scripts/celigo_api.py integrations flows 68e971adaf39cdcdec6ae854

# Rocky Brands flows
python3 scripts/celigo_api.py integrations flows 6807e03e7d181f7c98855103
```

### Run Specific EDI Flow
```bash
# Run Academy 850 inbound flow
python3 scripts/celigo_api.py flows run 68e971f853e4a108b0046436

# Run Rocky Brands 856 outbound flow
python3 scripts/celigo_api.py flows run 6807e074ea88c4cb9452c44e
```

## Notes

- **Total**: 29 production EDI integrations (25 trading partners + 4 infrastructure)
- **Dates in names**: Indicate production deployment date
- **VAN prefix**: Trading partner uses Value Added Network (York, ECGrid)
- **SPS suffix**: Uses SPS Commerce as EDI provider
- **Excludes**: Sandbox integrations (sandbox: true)

## Maintenance

This list should be updated when:
- New trading partners go to production
- New EDI documents are added to existing partners
- Integrations are retired or renamed
- Integration IDs change (rare)

To refresh integration list:
```bash
python3 scripts/celigo_api.py --format json integrations list | \
  python3 -c "import json,sys; [print(f\"{i['_id']}|{i['name']}\") for i in json.load(sys.stdin) if 'EDI' in i.get('name','') and not i.get('sandbox',False)]"
```
